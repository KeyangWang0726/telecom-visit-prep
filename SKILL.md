# 电信走访通

为中国电信客户经理提供一键式企业走访准备服务：输入企业名称，自动完成企业信息搜索、商机智能推荐、拜访话术生成，输出完整走访准备报告。

## 前置依赖检查

本技能依赖 `docx` skill 生成Word报告。在执行工作流程之前，先检查依赖：

- 检查 `docx` skill 是否已安装（检查 `~/.config/teleai-super-agent/skills/docx/SKILL.md` 是否存在）
- 如不存在，执行安装：从技能市场下载 `docx` skill 到 `~/.config/teleai-super-agent/skills/`
- 安装失败时在对话中提示用户手动安装，但不阻塞后续流程（报告仍可在对话中正常输出）

## 工作流程

```
依赖检查 → 输入企业名称 → 搜索企业信息 → 生成客户画像 → 匹配商机线索 → 制定走访计划 → 生成沟通方案 → 对话中输出报告
```

严格按以下5个步骤顺序执行：

### 第1步：多维度企业信息搜索

使用 `baidu_search` 工具进行多轮搜索，全面收集企业信息。搜索维度和策略：

| 搜索轮次 | 搜索关键词模板 | 信息目标 |
|---------|--------------|---------|
| 第1轮 | `"{企业全称}" 工商信息 简介` | 基本工商信息、成立时间、注册资本、经营范围 |
| 第2轮 | `"{企业全称}" 新闻 最新动态` | 近期业务动态、新闻事件、发展规划 |
| 第3轮 | `"{企业全称}" 公众号 抖音 官网` | 企业官方渠道、品牌形象 |
| 第4轮 | `"{企业名称}" {所属行业} 数字化 转型` | 行业趋势、信息化需求线索 |

搜索策略：
- 每轮使用不同的关键词组合，确保覆盖面
- 企业全称指用户输入的名称，如搜索结果有限则尝试使用简称或行业+地区组合搜索
- 重点关注：员工规模、业务扩张、信息化投入、招投标动态、上下游合作关系
- 每轮搜索最多2次（同轮内并行），共4轮不超过8次搜索
- 搜索结果中注意识别：负面舆情（法律纠纷、经营异常）、正面亮点（荣誉资质、行业地位）

### 第2步：整合企业信息，构建客户画像

将搜索到的信息整合为结构化画像：

**必需字段**（尽力填充，缺失标注"未获取到"）：
- 企业全称、简称、所属行业、成立时间、注册资本、员工规模
- 主营业务、业务模式（B2B/B2C/混合）、行业地位
- 注册地址、办公地址（如不同）、分支机构

**关键分析维度**：
- **企业规模判断**：根据注册资本、员工人数、营收规模判断大/中/小微型
- **信息化水平推测**：根据行业属性、搜索到的IT相关新闻推测
- **业务增长趋势**：根据近期新闻、招聘动态、扩张信息判断
- **通信需求信号**：识别是否有远程办公、多地协同、大量内部沟通等需求

### 第3步：智能商机推荐

读取 [references/telecom-products.md](references/telecom-products.md) 获取完整产品知识库，基于客户画像进行商机匹配：

匹配逻辑：
1. 根据"行业属性"匹配行业解决方案
2. 根据"规模特征"匹配适配产品档次（基础版/标准版/高级版）
3. 根据"业务特征"匹配具体产品（如多地办公→SD-WAN，制造业→5G+工业互联网）
4. 根据"信息化水平"推荐进阶产品（如低信息化→先推基础通信+云，高信息化→推AI+安全）

输出要求：
- 推荐3-5个产品/服务方案，按优先级排序
- 每个推荐必须结合该企业的**具体信息**给出推荐理由，禁止套话
- 如该企业有特殊的行业特性或业务模式，优先推荐与之强相关的方案

### 第4步：制定走访计划

基于客户画像和商机分析，制定结构化走访计划（6大模块）：

1. **走访目标设定**：明确本次走访要达成的结果层级（建立关系 / 摸底需求 / 推进方案 / 促成签约），根据企业信息成熟度判断目标层级
2. **走访节奏安排**：规划分阶段拜访节奏（首次拜访→二次回访→方案汇报），每阶段的目标和具体行动
3. **切入话题与话术顺序**：设计从破冰到推进的完整话术链路（破冰→需求探寻→产品引入→深度推介→促成下一步），明确每个阶段说什么、怎么过渡
4. **拜访前准备清单**：需要携带的资料、提前了解的行业信息、需要谁陪同、预约注意事项
5. **风险评估与预案**：可能遇到的困难（决策人不接触/竞对在场/客户无预算）、竞对情况分析、应对策略
6. **跟进计划**：走访后X天内做什么、下次拜访时机与目标、客户分层归属判断

### 第5步：生成定制沟通方案

读取 [references/speech-scripts.md](references/speech-scripts.md) 获取话术模板库，基于企业实际情况和走访计划生成沟通方案：

生成内容：
- **破冰开场话术**：必须提及该企业的具体信息（如近期新闻、行业成就），体现"做过功课"
- **需求探寻话术**：根据推荐的产品方向，设计3-5个针对性问题
- **产品推介话术**：选取优先级最高的2-3个产品，生成定制推介话术
- **过渡与推进话术**：各阶段之间的过渡语和推进话术（从探寻自然引入产品介绍、从介绍推进到促成下一步）
- **异议应答预案**：根据行业特征推测可能异议并准备应答
- 每类话术提供**商务正式版**和**亲和自然版**两个版本

### 第5步：在对话中输出完整报告

读取 [references/report-template.md](references/report-template.md) 获取输出模板，严格按模板结构组织报告内容。

**对话中依次输出**（不生成文件）：
1. **摘要**：企业定位（行业+规模+主营业务）、核心商机列表（3-5项）、建议切入点
2. **完整报告**：按模板结构依次输出企业概况、商机分析、拜访话术、走访建议全部章节

**末尾询问**：
输出完报告后，向用户确认："是否需要将此报告导出为Word文档？如需要，请回复"需要"，我将为您生成 .docx 文件。"
- 用户确认需要时，调用 `docx` skill 生成Word文件，命名规则 `{企业名称}走访准备报告.docx`，保存至工作目录根目录
- 用户不需要则结束流程

**报告内容规范**：
- 报告整体以客户经理第一人称视角呈现
- 企业信息标注来源（如"来源：企查查"、"来源：企业公众号"等）
- 商机推荐结合企业实际信息，理由具体不泛泛
- 话术体现针对性，有"做过功课"的质感
- 信息不足处如实标注"建议拜访中进一步了解"

## 关键注意事项

- 本技能适用于中国电信政企客户经理的日常走访准备工作
- 搜索结果可能存在信息滞后或不完整，报告中需提醒客户经理以最新信息为准
- 商机推荐仅供参考，实际方案需根据客户经理与客户的深入沟通后确定
- 对外资料不出现"出售海外模型""Claude/Gemini售卖"等字样
- 产品推荐以国内产品为主（天翼云、专线、V网等），星辰爱算平台如涉及海外模型一律以"高端模型扩展能力"表述

## 跨平台适配说明

本技能最初为星辰超级智能体开发，如需迁移到其他智能体平台（Claude Code、Codex CLI 等），需注意以下核心依赖的替换：

### 依赖项与替代方案

| 依赖项 | 依赖级别 | 星辰超级智能体原生 | 跨平台替代方案 |
|--------|---------|------------------|--------------|
| 网络搜索 | **核心** | `baidu_search`（内置） | MCP 搜索服务器 / Shell + 搜索 API / 手动提供信息 |
| Word 生成 | 非核心 | `docx` skill | python-docx 脚本 / MCP 文档服务器 / Markdown 输出 |
| 文件读取 | 核心 | 内置 | 各平台均内置，路径约定不同需注意 |
| 知识库引用 | 核心 | `[references/xxx.md](references/xxx.md)` | 不支持文件引用的平台需将 references/ 内容内联 |

### 必须修改的位置

迁移到其他平台时，需修改 SKILL.md 中的以下内容：

1. **前置依赖检查章节**：删除星辰平台特有的 `~/.config/teleai-super-agent/skills/` 路径检查，改为目标平台的对应检查方式
2. **第1步搜索**：将 `baidu_search` 替换为目标平台的搜索工具（如 Claude Code 的 MCP 搜索）
3. **报告输出**：将 `docx` skill 调用改为 python-docx 脚本或其他文档生成方式

### 详细适配指南

完整的跨平台适配步骤和各平台具体配置方法，请参阅 [README.md](README.md) 中的「跨平台适配指南」章节。

### 兼容性速查

| 平台 | 适配难度 | 备注 |
|------|---------|------|
| Claude Code | 低 | Skill 格式兼容，仅需替换搜索和文档生成工具 |
| Codex CLI | 中 | 无 Skill 系统，需内联知识库，通过 AGENTS.md 注入 |
| Cursor / Windsurf | 中 | 通过 .cursorrules 注入，需 MCP 补充搜索能力 |
